# Multiple Choice Question (MCQ) Feature

## Overview
This feature introduces Multiple Choice Questions (MCQs) for **Medium** and **Hard** difficulty levels across all question categories. **Easy** difficulty remains as open-ended text input.

## Implementation Details

### Question Generation
- The `Question.generate()` method signature has been updated to return a tuple: `(question_text, answer, choices)`.
- `choices` is a list of 4 options (strings or numbers) for Medium/Hard difficulties.
- `choices` is `None` for Easy difficulty.
- A helper method `generate_choices(correct_answer, is_integer)` in `Question` base class generates 3 distractors based on the correct answer.

### Distractor Logic
- **Integers**: Distractors are generated by adding/subtracting small random integers (1-10) to the correct answer.
- **Decimals**: Distractors are generated by adding/subtracting small random floats (0.1-5.0) to the correct answer.
- Choices are always shuffled.

### Console Interface (`math_game.py`)
- Detects if `choices` is present.
- Displays options as A, B, C, D.
- Accepts input as 'A', 'B', 'C', 'D' (case-insensitive).
- Maps user choice back to the actual value for validation.

### Web Interface (`app.py` & `game.html`)
- Passes `choices` to the template.
- `game.html` renders a list of buttons for choices if available.
- Each button submits the corresponding value as the answer.
- Existing text input is displayed if `choices` is `None` (Easy mode).

## Affected Files
- `questions/base.py`: Added `generate_choices` method.
- `questions/*.py`: Updated `generate` method in all categories.
- `math_game.py`: Updated game loop for MCQ handling.
- `app.py`: Updated route to handle choices.
- `templates/game.html`: Added MCQ button rendering.
- `test_math_game.py` & `test_app.py`: Updated tests for new method signature.
